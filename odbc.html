ODBC Data Access

Open Database Connectivity Support

Our data system utilizes a NoSQL database as storage layer, however, it's possible to use a universal ODBC connector; this will allow you to connect your favorite graphic data explorer tool directly to our datasources.

You can learn more about ODBC here.

Connection Procedure

  1. Verify the system requirements below!
  2. Install the required driver for your Operating System
  3. Configure the ODBC source; for more information please consult the reference
  4. Connect your favorite application to the existing datasource; connections details will be provided during the event!

System Requirements

Please ensure your equipment meet the appropiate system requirements based on the OS you're using.

  • Windows
  • Mac OS
  • Linux
  • The 64-bit editions of the following operating systems are supported:
    • Windows® 8 and 8.1
    • Windows® 7 Professional
    • Windows® Server 2008, 2013 R2
  • .NET Framework 4.5, installed and enabled by default on Windows 8 and later
  • 60 MB of available disk space
  • Install the driver available here
  • Mac OS X version 10.6.8 or later
  • 100 MB of available disk space
  • iODBC 3.52.7 or later
  • Install the driver available here
  • One of the following distributions (32- and 64-bit editions are supported)
    • Red Hat® Enterprise Linux® (RHEL) 5.0/6.0
    • CentOS 5.0/6.0
    • SUSE Linux Enterprise Server (SLES) 11
  • 90 MB of available disk space.
  • One ODBC driver manager installed
    • iODBC 3.52.7 or above
    • unixODBC 2.2.12 or above
